# Autoresearch: Domain Support Optimization

This is an autonomous optimization skill generated by autoresearch.
It runs an eval-driven loop that repeatedly improves code against
binary pass/fail metrics.

## Goal

Optimize domain support coverage and quality across all 6 autoresearch strategy types
(coverage-improvement, performance-optimization, test-hardening, code-quality,
feature-completion, custom).

## Architecture

### Fixed Foundation (DO NOT MODIFY during loop)

- This skill directory (except state.json and results/)

## Loop Protocol

Follow this protocol autonomously without pausing for user input.

### State Machine

INIT → ANALYZE → HYPOTHESIZE → MODIFY → EVALUATE → DECIDE → [ANALYZE | DONE]

### ANALYZE
1. Read the latest results (baseline or last iteration) from `results/`
2. Identify failing evals from eval.json
3. Read target files and error output
4. Check hypothesis log in state.json for previously tried approaches

### HYPOTHESIZE
1. Form a specific, testable hypothesis
2. Verify it hasn't been tried before (check state.json)
3. Log the hypothesis to state.json before making changes

### MODIFY
1. Verify all files are within target_files bounds
2. Verify no files are in protected_files
3. Make minimal, focused changes implementing the hypothesis
4. Verify changes are well-formed markdown or valid JSON

### EVALUATE
1. Run: `python3 .claude/skills/autoresearch-domain-support/scripts/eval_runner.py <iteration>`
2. Collect pass/fail results for each eval
3. Save results to `results/iter-<NNN>.json`

### DECIDE
- **Improvement** (more evals pass): `git commit` with message format:
  `autoresearch(domain-support): iter <N> — <hypothesis summary>`
- **Regression** (fewer evals pass): `git checkout -- <target_files>`
- **Neutral** (same results): `git checkout -- <target_files>`

### Termination
Stop when: all evals pass OR iteration >= max_iterations OR plateau detected.

## Strategy: Custom

Improve domain support by:
1. **Expand custom strategy** — Add structured sections (Analysis approach,
   Hypothesis patterns, Common pitfalls) to the custom strategy in
   domain-strategies.md
2. **Add concrete glob patterns** — Replace descriptive text with actual
   file glob patterns in strategies that lack them (performance-optimization,
   test-hardening, code-quality, feature-completion)
3. **Expand eval test cases** — Add test cases to evals.json for
   underrepresented domains (code-quality, feature-completion)
4. **Ensure consistency** — Keep strategy names and references aligned
   across all files

## Safety Rules

1. NEVER modify eval.json, eval_runner.py, validate_domains.py, or this SKILL.md during the loop
2. ONLY modify files matching target_files in eval.json config
3. NEVER modify files matching protected_files in eval.json config
4. Commit improvements, revert regressions — no exceptions
5. Log every hypothesis before modifying code
6. Stop at max_iterations even if goals aren't met
7. Stop if no progress for 5 consecutive iterations
8. No interactive prompts during the loop
9. All work on experiment branch — never modify main/master
